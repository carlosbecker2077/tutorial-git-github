Tutorial de Git e Github

Git e Github são coisas diferentes. Git é a ferramenta de versionamento local e Github é o repositório/rede social.

Tem dois jeitos de fazer o negócio funcionar, com o programinha desktop e sem. Com o programinha é bem barbada e mais fácil de controlar as coisas, além de ter mais opções visuais e de integração com o Windows (O github também é da Microsoft agora). Sem o programinha é tudo direto no terminal do Git, o Git Bash. Escolhi fazer com o programinha já que acredito que tu não tá mais na fase de aprender um monte de código só pra se aparecer, e também acredito que é mais fácil mesmo.

1. Instalação das coisas

Será necessário fazer o download das seguintes coisas:
  • Git:Download Git para Windowns 64bit
  • GitHub Desktop:Download GitHub Desktop para Windows 64Bit
  • Visual Studio Code: Pode ser útil ter, já que o GitHub Desktop tem integração com ele. Mas não precisa baixar se não quiser, não afeta no processo. Download Visual Studio Code

Feito isso agora tá na hora de instalar as coisas

1.1 Instalação do VS Code

É bem direto, a única coisa é marcar as duas opções que tá escrito "Open with Code" que é pra poder clicar com o botão direito do mouse e abrir a pasta com o Code.

Já com o VS Code instalado dá pra fazer umas "melhorias" que também não são obrigatórias mas que eu gosto porque deixam a minha vida mais fácil, e é bom tu saber que tem mesmo que não queira usar.

  • Primeira coisa é ir em File > Auto Save para salvar automaticamente a cada coisa que for editada.
  • Depois ir em File > Preferences > Settings e na barra de pesquisa escrever "Word Wrap" e ligar. Isso serve para quebra de linha automática.
  • No canto esquerdo tem um ícone que parece um Tetris, é aqui que se adiciona as extensões e suporte a outras linguagens. Dá pra instalar suporte a Pascal se quiser. Digitando Pascal, é o primeiro.

1.2 Instalação do Git

É bem direto, só tem que mudar o Default Editor para o VS Code caso tu tenha instalado, o resto é só deixar como tá mesmo. Sempre ir nas opções com "Recomended".

1.3 Instalação do GitHub Desktop

Consegue ser mais fácil ainda, no que tu clicar para executar ele já vai instalar (e colocar um ícone na área de trabalho que eu detesto). Assim que tu logar com a tua conta do GitHub, ele vai pedir o teu nome (que eu acho que já vai trazer o que tá na conta) e o email. O email que vem ali é do tipo "noreply" para caso tu não queira divulgar o teu email para o público, mas dá pra mudar depois. Depois disso ir em File > Options > Appearance > Dark Mode :)

Feito! Agora dá pra começar a usar esta bagaça.

2. Usando esta bagaça

2.1 Pegando Repositórios (Pull e Clone)

O Pull "puxa" os repositórios que estão no Github pro teu PC. Serve para atualizar os fontes. Já no caso de de pegar o reposotório pela primeira vez, será um Clone. Eu sei, é praticamente a mesma coisa, mas só para diferenciar as duas coisas.

Melhor demonstrar com um exemplo né? Então faz assim:

  1. Entra aqui nesse cara, que é o meu repositório, mais precisamente o repositório desse tutorial;
  2. Clicar no botão verde escrito "Code";
  3. Abrir com GitHub Desktop;
  4. Escolher onde salvar;
  5. E era isso!

2.2 Atualizando os repositórios (Fetch)

O Fetch é basicamente para verificar se tem alguma atualização no repositório. Sempre que eu ou tu formos mexer em algo, fazer primeiro o fetch para ver se não há nenhuma modificação.

Para fazer o fetch:

  1. Logo abaixo de File tem um menu dropdown "Current repository", aqui é onde tu vai poder navegar entre os repositórios;
  2. Seleciona o repositório que tu quer;
  3. E clica em fetch, que vai ser a última opção na mesma linha do "Current repository";
  4. E agora o teu projeto está atualizado.

2.3 Registrando um repositório

  1. Ir em File > New Repository;
  2. Colocar o nome (Ex o nome da empresa);
  3. Indicar o caminho de onde será criada a pasta do repositório;
  4. Inicializar com o README.md é opcional;
  5. Selecionar Delphi ou a linguagem a ser tratada no Git Ignore;
    O Git Ignore serve para ignorar os arquivos binários e também é possível adicionar outros arquivos a serem ignorados como .pdf, .jpg, .conf...
  6. Acredito que a licença seja só para o caso de disponibilizar o projeto para o público. Deixa sem;
  7. E "Create Repository".

2.4 Fazendo o Commit

Importante ressaltar que commit e push são coisas diferentes. O commit versiona os arquivos localmente no teu computador, enquanto o push é o que publica no GitHub. É bem estranho no início já que no SVN é o commit que faz a mágica acontecer, então por vezes tu vai fazer o commit achando que foi, mas na real vai estar faltando o push para que o projeto seja realmente "empurrado" para o GitHub.

  1. Copiar todos os arquivos que estão na pasta do projeto para a pasta recém criada;
  2. Os arquivos copiados agora aparecerão no Git Desktop como "Untracked Files";
  3. Dá uma olhada nos arquivos adicionados e procura algum arquivo que tu não queira enviar, como um .pdf ou um .rar;
    • Se for um arquivo específico para ser ignorado, clica com o botão direito no arquivo e "Add file to .gitignore";
    • Se for uma extensão inteira de arquivo a ser ignorado, clica com o botão direito e "Add file extension to .gitignore";
  4. Escrever um nome para o commit, geralmente vai ser algo do tipo "Incluido campo tal", mas se for o primeiro commit pode ser "Initial commit" ou "Primeiro commit";
  5. E commitar!

2.5 Fazendo o Push

  1. Se for a primeira vez, "Publish repository". Se for qualquer outra vez "Push" ou a combinação de teclas "Ctrl + P"
  2. Colocar o nome que vai aparecer no GitHub;
  3. Manter privado ou não (o GitHub permite até 3 pessoas colaborando em um projeto privado, se for publico é ilimitado)
  4. Escolher uma organização
    • A organização serve para que tu não crie o repositório na tua conta, mas na da empresa. É recomendável a criação de uma organização para que fique tudo mais centralizado. Dá pra criar uma Provisual ou publicar tudo no Inforserv que já existe. Fica pra ti essa opção.
    • A organização vai aparecer ali se a tua conta for associada a uma
  5. Clicar em "Publish"
  6. E é isso! Agora dá pra ir lá no teu GitHub pra ver se tá lá.
